# JVM的四种引用
~~~
(1) 强引用
    不会被垃圾回收
    强引用是造成Java内存泄露的主要原因之一
(2) 软引用
    内存充足时   不会被回收
    内存不足时    会被回收
    通常用在对内存敏感的程序中,比如高速缓存
(3) 弱引用
    WeakReference
    内存不足时    会被回收
(4) 虚引用
    PhantomReference
    与其他几种引用都不同,虚引用并不会决定对象的生命周期
    任何时候都可能被垃圾回收器回收,它不能单独使用也不能访问对象,虚引用必须和引用队列(ReferenceQueue)联合使用

    主要是跟踪对象垃圾回收的状态,做垃圾回收之前的清理操作
~~~
